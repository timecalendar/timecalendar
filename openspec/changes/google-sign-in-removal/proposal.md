## Why

Phase 2 ([TIM-3](/TIM/issues/TIM-3)) is the dependency-upgrade & maintenance
phase. The B1 audit ([TIM-36](/TIM/issues/TIM-36)) called out the
`google_sign_in` Flutter plugin as a candidate for a 6→7 major bump in B6
([TIM-48](/TIM/issues/TIM-48)). The TIM-91 Plan re-audit found `google_sign_in`
is a **zombie dependency**: declared in `app/pubspec.yaml` at `^6.1.4` but
not used anywhere in Dart code (0 `package:google_sign_in/` imports across
`app/lib/`, `app/test/`, `app/integration_test/`). There is no auth screen —
TimeCalendar is anonymous calendar-sync (paste a schedule URL) — and the
server has no Google OAuth path either.

So the 6→7 migration is moot: the right action is to delete the dep. This
also drops a transitive native surface (`GoogleSignIn`, `AppAuth`,
`GTMAppAuth`, `GTMSessionFetcher` iOS frameworks, plus the
`google_sign_in_android` plugin), shrinks `Podfile.lock`, and clears a dead
OAuth URL scheme from `app/ios/Runner/Info.plist`.

## What Changes

- **Remove the direct dependency** `google_sign_in: ^6.1.4` from
  `app/pubspec.yaml`. `pubspec.lock` is regenerated by `flutter pub get` and
  drops `google_sign_in`, `google_sign_in_android`, `google_sign_in_ios`,
  `google_sign_in_platform_interface`, and `google_sign_in_web`.
- **Drop the dead OAuth URL scheme** `com.googleusercontent.apps.425450003183-1elevc6cg39idiukk9lh5sjtufis8h0r`
  from the `CFBundleURLTypes` array in `app/ios/Runner/Info.plist`. This
  scheme is the OAuth callback for `google_sign_in`'s `REVERSED_CLIENT_ID`
  and is dead with the plugin gone. The Facebook scheme
  `fb2272446229720075` in the same array stays.
- **Surgically remove the Pods-managed entries** for `google_sign_in_ios`,
  `GoogleSignIn`, `AppAuth`, `GTMAppAuth`, and `GTMSessionFetcher` from the
  `[CP] Copy Pods Resources` and `[CP] Embed Pods Frameworks` script phases
  in `app/ios/Runner.xcodeproj/project.pbxproj` (10 input/output path
  entries). CocoaPods would normally rewrite these on a Mac `pod install`;
  on the Linux Applier host the edit is mechanical and idempotent (delete
  exactly those 10 lines).
- **Keep** `app/ios/Runner/GoogleService-Info.plist` (`CLIENT_ID` /
  `REVERSED_CLIENT_ID` and the rest) **untouched**. Those keys are emitted
  by the FlutterFire CLI alongside Firebase config; they coexist with
  Google Sign-In but are not exclusive to it, and removing them risks
  surprising Firebase. Documented keep decision in `design.md`.

Out of scope: any new sign-in feature, any Firebase Auth refactor (the
`firebase_auth` dep is part of the FlutterFire v4 suite locked in
[TIM-87](/TIM/issues/TIM-87) and moves with the suite — not in this change),
any Android Gradle / Java edits (the Android plugin registration is in the
auto-generated, git-untracked `GeneratedPluginRegistrant.java` and
regenerates from pubspec).

### Modified Capabilities
- `flutter-dependency-hygiene`: adds a requirement that `google_sign_in`
  is removed as a zombie dep, alongside the native-side cleanup
  (Info.plist URL scheme + pbxproj script-phase entries) that follows
  from removing a Flutter plugin with iOS pods. Does not alter the
  existing B3 requirements about other dead/stale packages.

## Impact

- `app/pubspec.yaml` — `google_sign_in: ^6.1.4` removed from `dependencies:`.
- `app/pubspec.lock` — regenerated by `flutter pub get`; five
  `google_sign_in*` packages drop out.
- `app/ios/Runner/Info.plist` — the dead `com.googleusercontent.apps.…` URL
  scheme is removed from the `CFBundleURLTypes` array.
- `app/ios/Runner.xcodeproj/project.pbxproj` — 10 stale Pods-managed
  input/output path entries removed (1 privacy bundle pair + 4 framework
  pairs) from the `[CP] Copy Pods Resources` and `[CP] Embed Pods
  Frameworks` script phases.
- `app/ios/Podfile.lock` — regenerated by `pod install` in CI's iOS build
  job; `google_sign_in_ios`, `GoogleSignIn`, `AppAuth`, `GTMAppAuth`,
  `GTMSessionFetcher` Pods drop out. (`Podfile.lock` is git-ignored, no
  on-disk delete needed.)
- `app/ios/Runner/GeneratedPluginRegistrant.m` and
  `app/android/app/src/main/java/io/flutter/plugins/GeneratedPluginRegistrant.java`
  — regenerated by `flutter pub get`. Both files are git-untracked, so they
  are not part of the diff.
- No Dart source-code edits (0 call sites). No backend / `openapi/` / `web/`
  / CI / SDK-constraint changes. Behaviour is unchanged: no feature uses
  Google Sign-In today.
